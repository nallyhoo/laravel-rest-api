# Professional RESTful API Development Guide (Laravel 11+)

This guide outlines the step-by-step process used to build the **Project Inventory Management System**. It follows industry best practices for scalability, security, and performance.

## 1. Project Initialization
Start by creating a new Laravel project and setting up the API scaffolding.

```bash
# Create project
composer create-project laravel/laravel my-api-project

# Install API support (Required for Laravel 11+)
php artisan install:api
```

## 2. Database Schema & Models
Use migrations to define the structure and Eloquent models for data interaction.

### Create Entities
```bash
# -a creates Migration, Factory, Seeder, and Controller
php artisan make:model Project -a --api
php artisan make:model Product -a --api
```

### Define Relationships
In `app/Models/Project.php`:
```php
public function products() {
    return $this->hasMany(Product::class);
}
```

In `app/Models/Product.php`:
```php
protected $fillable = ['name', 'sku', 'price', 'stock', 'project_id'];

public function project() {
    return $this->belongsTo(Project::class);
}
```

## 3. Robust API Controllers
Controllers should handle the logic for GET, POST, PUT, and DELETE requests.

### Key Implementation (ProductController.php)
*   **Index**: Use `with()` for eager-loading to avoid N+1 query problems.
*   **Store/Update**: Use Form Requests for validation.
*   **Response**: Return JSON with appropriate HTTP status codes (e.g., `201` for Created).

```php
public function index() {
    return Product::with('project')->get();
}

public function store(StoreProductRequest $request) {
    $product = Product::create($request->validated());
    return response()->json($product, 201);
}
```

## 4. Validation (Security)
Never trust user input. Use **Form Requests** to centralize validation logic.

```bash
php artisan make:request StoreProductRequest
```

Example rules:
```php
return [
    'name'  => 'required|string|max:255',
    'sku'   => 'required|unique:products,sku',
    'price' => 'required|numeric|min:0',
    'stock' => 'required|integer|min:0',
    'project_id' => 'required|exists:projects,id'
];
```

## 5. API Routing
Define your routes in `routes/api.php` using `apiResource` for standard REST patterns.

```php
use App\Http\Controllers\ProductController;
use App\Http\Controllers\ProjectController;

Route::apiResource('projects', ProjectController::class);
Route::apiResource('products', ProductController::class);
```

## 6. Frontend Integration (UI)
Connect your UI to the API using **Axios**.

### The Setup
1.  **Axios Config**: Create a helper to set `Accept: application/json` headers.
2.  **State Management**: Maintain a local `state` object to keep the UI in sync without full page reloads.
3.  **Dynamic Rendering**: Use modern CSS (Flexbox/Grid) and icons (Lucide/FontAwesome) for a premium feel.

### Example Fetch
```javascript
async function fetchProducts() {
    const res = await axios.get('/api/products');
    state.products = res.data;
    renderUI(); 
}
```
