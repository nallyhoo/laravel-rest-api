## ðŸ“¡ 4. Data Flow Patterns

### A. Fetching Data (Async/Await)
We use `async/await` for clean, readable code. Notice how we handle both "All Products" and "Project Specific" items in one function:

```javascript
async function fetchProducts(projectId = null) {
    const url = projectId ? `/projects/${projectId}` : '/products';
    const res = await api.get(url);
    
    // If we fetch a single project, its products are in res.data.products
    state.products = projectId ? res.data.products : res.data;
    renderProducts();
}
```

### B. Dynamic Rendering
Instead of static HTML, we use template literals to inject rows based on the API response:

```javascript
function renderProducts() {
    tbody.innerHTML = state.products.map(product => `
        <tr>
            <td>${product.name}</td>
            <td>$${product.price}</td>
            <td>${product.stock} in stock</td>
        </tr>
    `).join('');
}
```

## ðŸ“ 5. Handling Forms
We intercept standard HTML form submissions to prevent page reloads.

```javascript
window.handleProductSubmit = async (e) => {
    e.preventDefault(); // Stop page from refreshing
    const formData = new FormData(e.target);
    const data = Object.fromEntries(formData.entries());

    await api.post('/products', data); // Send to API
    fetchProducts(); // Refresh the list
    closeModal();    // Hide the popup
};
```
