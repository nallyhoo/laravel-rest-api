## ğŸ’¡ Tips for Better Testing

### 1. Environments
Instead of typing `http://127.0.0.1:8000/api` every time:
1. Create a new **Environment** in Postman.
2. Add a variable named `base_url` with the value `http://127.0.0.1:8000/api`.
3. Use `{{base_url}}/products` in your request URL.

### 2. Status Codes to Watch For
*   `200 OK`: Request succeeded.
*   `201 Created`: Resource successfully created (Post/Store).
*   `204 No Content`: Resource deleted successfully.
*   `422 Unprocessable Content`: Validation failed (check your SKUs or IDs).
*   `404 Not Found`: The Project or Product ID does not exist.
